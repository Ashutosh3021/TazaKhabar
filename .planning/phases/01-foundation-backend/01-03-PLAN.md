    1. **src/services/trend_service.py:**
    - TECH_KEYWORDS list (50+ common tech keywords for counting — see interfaces block above)
    - TrendService class:
      * async def extract_keywords(text: str, keywords: list[str]) -> set[str] — tokenize text (lowercase, strip punctuation, split on whitespace), return matching keywords found
      * async def compute_keyword_frequencies(session: AsyncSession) -> list[Trend] — TRND-01 compliance:
        - Query all jobs and news with report_version="1"
        - For each job: extract keywords from job.title + " ".join(job.tags) + job.summary
        - For each news: extract keywords from news.title + news.summary
        - Count frequency of each keyword across all items
        - Determine week_start (7 days ago, start of day), week_end (now)
        - Get previous week's count for each keyword from trends table (week_start exactly 7 days before)
        - Compute percentage_change = ((current_count - previous_count) / previous_count * 100) if previous_count > 0 else 100.0
        - Classify: if percentage_change > 20 → booming, if percentage_change < -20 → declining, else neutral
        - Upsert into trends table
      * async def get_trends(session: AsyncSession, limit: int = 20) -> list[dict] — TRND-06 compliance:
        - Query trends table for current week (most recent week_start)
        - Sort by percentage_change desc, return top 5 booming
        - Sort by percentage_change asc, return top 3 declining
        - Map to TrendResponse: {skill, percentage (abs value), weeklyChange (+/-)}
    - Add APScheduler job: run compute_keyword_frequencies every 24 hours (weekly computation)

    2. **src/api/trends.py** — FastAPI router for /api/trends:
    - GET /api/trends — returns {data: [Trend, ...], meta: {count: int}}
    - GET /api/trends/observation — returns {data: {text: string}} — for now return static placeholder: "Trends are being computed. Check back after your first scrape cycle." (TRND-08 — Phase 2 LLM narration deferred)
    - TRND-07 compliance: this endpoint replaces mock trends data

    3. **src/main.py update** — Include trends.router at /api/trends
